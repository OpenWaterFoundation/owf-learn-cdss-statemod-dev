Error Handling and Logging

StateMod, as a Fortran program, does not have extensive error-handling from language perspective. However, due to its complexity, many lines of code are devoted to checking input for calculations and handling appropriately. Output is to standard output and a log file.

The following sections discuss specific topics relevant to error handling and logging.

Error Handling for READ and WRITE Statements

The code does utilize the ERR=linenumber feature of READ and WRITE statements. This allows an error to trigger jumping to a line in the code, which can output an error message, for example.
